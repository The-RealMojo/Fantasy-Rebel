# Fantasy Soccer Game – MVP Definition

## Goal of the MVP
The goal of the MVP is to deliver a fully playable fantasy soccer game that demonstrates:
- Object-Oriented Programming in Java
- Clear separation of frontend, backend, and database
- Clean, maintainable architecture
- A polished, usable web interface

The MVP prioritizes **core gameplay** over advanced features.

---

## MVP Features (Must-Have)

### 1. User Management
- User registration
- User login
- One fantasy team per user

---

### 2. Fantasy Team Management
- Create a fantasy team
- Select 8 players within a £50m budget
- Assign starters and substitutes
- Select a captain

---

### 3. Scoring System
- Backend-based points calculation using OOP Java
- Support for:
    - Goals
    - Assists
    - Clean sheets
    - Cards
- Captain points multiplier

---

### 4. Gameweeks
- Manual or simulated gameweek processing
- Points calculated after each gameweek
- Team total points stored and displayed

---

### 5. Leaderboard
- Global leaderboard showing:
    - Team name
    - Total points
- Updated after each gameweek

---

### 6. Frontend Interface
- Built using HTML, CSS, JavaScript, React, and Tailwind
- Pages:
    - Login / Register
    - Team Builder
    - Dashboard
    - Leaderboard
- Responsive design for desktop and mobile

---

## Out of Scope for MVP (Planned for Future)
These features are intentionally excluded from the MVP to keep the project realistic and focused:

- Live match updates
- Player price changes
- Form indicators
- Fixture difficulty ratings
- Private leagues
- Transfer penalties
- Achievements and awards
- Real-world football APIs

---

## Success Criteria
The MVP is considered complete when:
- A user can create an account
- Build a fantasy team
- Earn points through backend logic
- View results on a leaderboard
- Use the application through a clean web interface
